Handlebars模板引擎常用语法总结

最近项目中用到了Handlebars，学习之余顺手写了一篇总结。

表达式

模板 + 数据 => (模板引擎) => html页面

handlebars中变量都添加双花括号来表示，访问json格式的数据可以使用.或者/, 其中变量名不可包含以下字符,如果包含则不被解析，但可以用[]来转义。

以上规则表明，handlebars是一种若逻辑模板引擎， “&&”,”||”,”!”这类逻辑判断是不能出现在表达式中的。 后面会有针对性解决方案。

帮助函数（helper）

handlebars本身提供了在模板中注入帮助函数的能力，你可以调用自带 或 自定义的helper函数, 传入参数并进行逻辑处理。

自带的helper

if else

用法与其他模板引擎大体无差，不过重申上面提到的特殊字符， if条件中 不能有逻辑表达式，只能是变量或者值。 如果想要实现比较指定值，比如 someNumber === 32 ,这种自带helper无法满足，需要使用后面介绍的自定义helper。

unless

默认用来处理逻辑非(!) 的语法

each

见名知意，用于处理遍历数据，相当于for循环， 多用于批量生成dom。

  • 提供默认变量
    • @first/@last 当该对象为数组中第一个/最后一个时返回真值
    • 当前遍历对象对数组时，@index表示当前索引，this表示当前值
    • 当前遍历对象为对象时，@key表示当前key, this表示当前值
  • 可以用相对路径的方式来获取上层的上下文，多用于多层遍历取外层遍历数据，用法见下例
  • 可以用 as |xxx|的形式给变量起别名，循环中通过别名可以引用父级变量值。当然也可以通过相对路径的方式引用父级变量。（不过亲测serv_shop项目自定义名会报错，原因未知。）

with

跟js中with含义差不多，用于更换当前作用域

自定义helper

当以上自带的helper无法满足需求的时候，比如someNum === 32，each的索引默认从零开始，现在需要从1开始，等等诸如此类的需求自定义helper你值得拥有，它可以让你用Handlebars.js来处理任何数据！

自定义helper分两类：行级helper, 块级helper。他们之间的一个显著区别就是helper函数体是否带有options参数。下面分别来详细介绍。

行级helper

行级helper多用于处理数据，类似于管道，将变量传入自定义函数最终按照一定逻辑输出。（试试多变量的情况）

块级helper

块级helper用于处理更加复杂的if判断逻辑，满足条件执行if区块，否则执行else区块。

以上就是Handlebars模板引擎的常见用法，欢迎指正，欢迎补充~
